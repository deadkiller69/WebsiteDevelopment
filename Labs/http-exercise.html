1. What is the Hypertext Transfer Protocol (HTTP)?

Hypertext Transfer Protocol(HTTP) is an application-layer protocol for transferring hypermedia documents, such as HTML, that follows a traditional client-server model and was developed for communication between web browsers and web servers, but may also be used for other applications.

2. Why is HTTP called a stateless protocol?

HTTP is called a stateless protocol because each command is executed independently, without any knowledge of the commands that came before it. Thus, the server does not store data between two requests.

3. Describe the three phases of establishing a session using client-server protocols such as HTTP.

The three phases of establishing a session using client-server protocols are:
  1. The client establishes a TCP connection.
  2. The client sends a request and waits for the answer.
  3. The server processes the request by sending back its answer, providing a status code and appropriate data.

4. Describe the parameters on a get and post request.

GET method is used to request data from specified resource.
POST method is used to send data to a server to create or update a resource.
The major distinction is that the GET method sends the request in the URL, whereas the POST method sends the data in the body part.

5. What are the HTTP response status code and What do the HTTP status codes 200, and 404 indicate?

HTTP response status codes shows whether or not a particular HTTP request was completed successfully.
200 indicates that the request has succeeded and 404 indicates file not found.

6. What are HTTP cookies? Give an example.

An HTTP cookie is a tiny piece of data sent by a server to a user's web browser. For example: it remembers stateful information for the stateless HTTP protocol.

7. Why are cookies a security issue?

Information is saved in cookies that is accessible to the user. If you're not careful, that information might be stolen by another user via third-party software.

8. Describe session hijacking and XSS?

Session hijacking is an attack in which the attacker takes control of a user's session without the user's knowledge, obtains the session ID, and uses it to impersonate someone else. XSS is a type of injection security attack in which an attacker injects data, such as a malicious script, into content from trusted websites.

9. Describe Cross-site request forgery (CSRF)?

Cross-site request forgery (commonly known as CSRF) is a web security vulnerability that allows an attacker to trick users into doing things they don't want to do.

10. What are Cross-site HTTP requests (CORS) and what requests use CORS?

CORS (Cross-Origin Resource Sharing) is an HTTP header-based method that allows a server to specify any other origins (domain, scheme, or port) from which a browser should permit resources loading. The CORS mechanism supports secure cross-origin requests and data transfers between browsers and servers.
